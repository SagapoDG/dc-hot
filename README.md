# DC·HOT — 数据合规与争议解决热点

聚焦**数据合规与争议解决**领域的国内外资讯聚合页面，逻辑参照 aihot.virxact.com：
多信源 RSS 聚合 → 双维度相关性闸门 → 自动分类 → 相似标题聚类去重（"n 个信源"）→ 热度排名（随时间消退）→ 卡片式榜单（精选/实时/全部动态，含分类筛选、国内/国际筛选、搜索、按日期分组、入选理由）。

**相关性闸门**：每条资讯必须同时命中「数据维度」×「法律维度」关键词，或命中仲裁/争议解决专门词才收录（专业信源通过 `implies` 预先满足相应维度），并剔除律所榜单类公关稿、限制单一外部媒体的条数。

**实时更新**：服务端每 10 分钟（`app.py` 里的 `REFRESH_MINUTES`）自动重抓全部信源；页面每 60 秒轮询，数据有更新即重新渲染并给新条目打 NEW 标；「实时」标签页完整展示近 48 小时全部资讯，统一按北京时间倒序（前端所有时间显示/分组均强制 Asia/Shanghai，与浏览器时区无关）。

**分类口径**：「立法监管」仅收录官方部门内容（须命中官方机构或法律文件名，见 `fetch_news.py` 的 `OFFICIAL_RE`），企业自身合规动态归入「行业实践」。

**要闻回顾**：「要闻」标签页汇总近两个月（`HIGHLIGHT_DAYS`）最重要的立法与案例，重要度 = 关键词分 + 信源权重 + 类别加成（立法监管/跨境数据 +3，诉讼仲裁/执法处罚 +2.5）+ 里程碑加成（`LANDMARK_RE`：首例/正式生效/创纪录罚款等 +3）+ 多信源加成；近似重复标题只保留最高分一条，每栏最多 `HIGHLIGHT_TOP_N`（默认 15）条。数据存于 60 天滚动档案 `archive.json`，每次抓取自动合并累积，并配有 `when:60d` 的 Google News 专用查询补足历史。

页面默认打开「实时」标签页。

纯 Python 标准库 + 单文件前端，无任何第三方依赖。

## 快速开始

```bash
cd dc-hot
python3 app.py          # 默认 http://localhost:8765 ，首次启动自动抓取
python3 app.py 9000     # 指定端口
```

页面右上角「↻ 刷新」按钮会触发服务端重新抓取全部信源。

也可以只生成数据不开服务：

```bash
python3 fetch_news.py   # 输出 news.json
```

## 文件结构

| 文件 | 说明 |
|---|---|
| `fetch_news.py` | 聚合脚本：信源配置、关键词词库、分类规则、热度算法都在这里 |
| `app.py` | 本地 HTTP 服务（页面 + `/api/news` + `/api/refresh`） |
| `index.html` | 前端页面（内嵌 CSS/JS） |
| `news.json` | 生成的数据文件 |

## 自定义

都在 `fetch_news.py` 顶部：

- **增删信源**：编辑 `SOURCES` 列表。`filter: True` 表示综合源（须命中强关键词才收录），`False` 表示专业源全收；`weight` 是信源权重。用 `gnews("关键词", zh=True/False)` 可直接添加一条 Google News 关键词检索源。
- **关键词**：编辑 `KEYWORDS`（按权重 3.0 / 2.0 / 1.0 分档，命中即加热度分）。
- **分类**：编辑 `CATEGORIES` 正则规则（按顺序先命中先归类）和 `REASONS` 入选理由模板。
- **时间窗口**：`MAX_AGE_DAYS`（默认只保留近 7 天）。

热度公式：`关键词分(≤12) + 信源权重 + 时效加成 + 2×(信源数-1)`，时效加成连续衰减（新发布 +4，72 小时后归零，即"热度随时间消退"）；全局热度 Top 10 标记 🔥 热点。

## 部署

- **本地定时更新**：crontab 里加 `0 * * * * cd /path/to/dc-hot && python3 fetch_news.py`，浏览器里点刷新即可看到新数据。
- **静态托管**：`index.html` 在取不到 `/api/news` 时会回退读取同目录 `news.json`，所以把 `index.html` + `news.json` 扔到任意静态空间（GitHub Pages、对象存储、Nginx）即可运行，配合 cron / GitHub Actions 定时重跑 `fetch_news.py` 更新数据。静态模式下「刷新」按钮不可用，其余功能不受影响。

## 已知限制

- 部分信源在国内网络环境可能无法直连（Google News、EDPB、FTC 等），抓取失败的源会自动跳过并在脚本输出中列出，不影响其余信源。
- 国内官方源（网信办等）普遍无 RSS，目前通过 Google News 中文关键词检索覆盖；如有自建 RSSHub 实例，可向 `SOURCES` 添加对应路由。
- 摘要与"入选理由"为规则自动生成，仅供参考，不构成法律意见。
